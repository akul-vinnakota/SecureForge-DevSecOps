# Security Policy

## Purpose

SecureForge is a defensive cybersecurity education and portfolio project.

## Reporting a Security Issue

Do not publicly disclose vulnerabilities involving real systems, credentials, accounts, or infrastructure.

Security findings related to this repository should be reported privately to the repository owner.

## Testing Restrictions

Testing must only be performed against:

- Systems owned by the project developer
- Deliberately vulnerable local applications
- Authorized lab environments
- Resources for which explicit permission has been granted

Do not use this project to access, scan, disrupt, or test systems without authorization.

## Sensitive Information

The following information must never be committed:

- API keys
- Passwords
- Access tokens
- Private keys
- AWS credentials
- Personally identifiable information
- Production configuration files

Demonstration credentials must be clearly fake and must not provide access to any real system.
